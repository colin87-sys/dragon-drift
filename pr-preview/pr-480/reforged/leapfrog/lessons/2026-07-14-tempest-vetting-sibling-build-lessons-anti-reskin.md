# Tempest §C — transfer LESSONS, never FEATURES, between sibling premiums (the anti-reskin triage)

**What we did.** Before cutting the Tempest build branch, a harsh Fable triage vetted all 15
Revenant build lessons (`2026-07-13-revenant-*.md`) for carryover into
`TEMPEST-THUNDERHEAD-BUILDSHEET.md` — now its `## §C`. Verdicts: **6 TRANSFER · 7 PARTIAL ·
2 REJECT**, each keeper REWRITTEN as a Tempest-specific directive and pinned to a §B increment,
plus an explicit ANTI-RESKIN GUARD (§C.3) with a standing gate veto.

**The problem this solves.** The owner's stated fear: dumping the Revenant's 15 lessons into the
Tempest sheet verbatim would make the builder produce a re-skinned Revenant. The two dragons are
structurally OPPOSITE — bone-lattice cage + lantern + bat-membrane vs opaque cloud decks + slit
gaps + intermittent lightning. Raw experience-dumping across siblings is how identity bleed
happens: the builder reaches for the proven pattern (a continuous frame, a membrane weld, a lit
cage) because a banked lesson praises it, and the new dragon quietly inherits the old dragon's
anatomy.

**The reusable pattern — the one-question triage.** For each lesson ask: *is this about how the
ENGINE / METHOD / CAMERA / MOTION / MATERIAL-under-light behaves — true whether the subject is
bone or cloud — or is it about a FEATURE of the previous creature?* Only the first kind
transfers, and even then it must be RE-DERIVED in the new dragon's vocabulary, never pasted:
- **Engine facts transfer whole** (the I0 builder contract, `coreGlow:null` stub, the
  horn/scales def gotcha, the byte-identity multiset proof, the flare-loop reset trap).
- **Camera/light/motion laws transfer as laws** (money-camera proportion; a hole/gap must FACE
  the cam, attitude before size; value laws are claims about game-lit pixels; distal amplitude <
  shoulder; a billboard is a sticker, a flat ribbon end-on is a wire).
- **Method transfers stripped of its subject** (waypoint-table profiles, shared-sample welding,
  pre-assess → build → gate, "grade articulated poses"). The Revenant welded MEMBRANE to finger
  spars; Tempest welds LIGHTNING polylines to deck seam samples — same method, zero shared
  geometry.
- **Feature lessons get REJECTED even when locally correct.** "An enclosed hole needs a
  continuous frame" is TRUE — and importing it would close Tempest's open-ended slit gaps into
  framed windows, i.e. build a ribcage into the wing. A correct lesson can still be poison in a
  different anatomy; the reject reason is identity, not validity.

**The gotchas worth paying forward:**
1. **A metric's pass-band can invert between siblings.** The Revenant's `holeMetric` climbs
   toward 26% enclosed holes; on Tempest, enclosed holes ≈ 0 is the PASS (its see-through is
   open-ended gaps). Same instrument, opposite polarity — so the sibling's tool transfers as an
   ANTI-reskin assert, which is the cheapest possible "not a re-skinned X" proof (the Revenant
   itself proved "not-Vesper" the same way: 93px vs 4px bays).
2. **Carryover triage catches latent spec bugs.** Holding the sheet against the Revenant's
   motion law exposed that §B.3b's dial block (`rootAmp .66, midAmp .28, tipAmp .42`) trips the
   broken-linkage tell (tip > mid; mid+tip 0.70 > root) — flagged for I2 re-tune before a single
   tri exists. A lessons audit is a free design review.
3. **Watch for superseded lessons in the pile.** `a-body-membrane-belongs-on-the-body` was
   corrected by the later pivot-anchor lesson; transferring both would import a known-wrong fix.
   Read the set as a NARRATIVE (later corrects earlier), not a flat checklist.
4. **Partial verdicts need an explicit banned half**, or the builder imports the whole file. Every
   §C PARTIAL names what transfers AND what is rejected ("light the receiver — but only in strike
   windows; continuous = the Grave-Heart lantern, banned").

**What it unlocks.** The Tempest builder starts with the Revenant's expensive process wins
(colour-render + Fable gate from increment ONE; byte-identity proof; money-camera tuning;
pre-assess → gate) WITHOUT its anatomy — and every future Fresh-Five build gets the same
one-question triage + a standing per-dragon reskin veto ("does any frame read like a re-skinned
<sibling> instead of a <identity>?") as the template for cross-pollinating parallel builds.
