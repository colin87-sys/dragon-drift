# Convert gate-DISCOVERIES into build PRE-conditions: the failure-mode firewall + round-0 self-audit

**What we did.** Reviewed the "before" render of the ORIGINAL Solar (the one the gate clawed back from):
spider-thin equal-weight spikes, ragged torn trailing edges, no wing mass, an insect/kite planform,
muddy leopard-spot skin, an empty rear-chase frame. Solar's Fable gate then climbed 1.10 → 4.19 over
~7 rounds and still needed a whole second campaign (CP2). The realization: those ~5 early rounds weren't
solving anything hard — they were REDISCOVERING nameable failure modes one at a time, because the sheet
let the builder negotiate the actual design AT the gate. Banked the fix into
[`PREMIUM-DRAGON-METHOD.md`](../../reforged/PREMIUM-DRAGON-METHOD.md) (new §3.5) and pre-scored the
Phoenix sheet (new QA-0).

**The reusable idea: every failure the gate is KNOWN to catch becomes a PRE-CONDITION the first build
must satisfy before it is allowed near a (high-effort, expensive) Fable round.** Three mechanisms:
1. **A failure-mode firewall** — the anti-patterns from the "before" image + the ledger, stated as hard
   invariants (confident faceted MASS not spider spars · a dominant element + scale hierarchy · a
   specific mass-bearing silhouette that reads at rear-chase, not a thin kite · something fills the rear
   frame · clean deliberate edges · opaque saturated bloom-safe emissive, no washout/olive-gold · no
   muddy texture · the spectacle triad present, not deferred).
2. **A round-0 SELF-AUDIT** — the implementing agent renders one capture and audits HARD against the
   firewall itself, fixing obvious violations, BEFORE spending a Fable round. Cheap self-gate first,
   expensive gate second → Fable never sees a build below ~3.5.
3. **A PRE-SCORED sheet** — the build sheet carries the firewall checklist + per-rubric-axis target
   scores + the reusable-kit inventory before a line of code. The design is decided on the sheet, not
   litigated at the gate. Result: the gate VERIFIES in 2–3 rounds instead of DISCOVERING over 7–10, and
   front-loading spectacle/rear-chase/silhouette means no inert-dragon CP2 second campaign.

**The critical framing trap we avoided (owner-flagged).** The tempting shorthand from the retro was
"firewall = the M-monarch silhouette, not a kite." That would push every future dragon toward Solar's
look — the exact sameness [§0.5 of the method] forbids and Phoenix (a not-M train-hero) violates. So the
firewall invariants are strictly **QUALITY/PROCESS floors** ("don't be thin / muddy / generic / washed
out / inert"), and every silhouette invariant is **shape-AGNOSTIC** (forbids the generic thin-spike
outline, never dictates a shape — the shape stays the dragon's own). "Reuse the KIT (quality floor), not
the LOOK" — you never start from the stick again, but you also never copy Solar.

**The high-effort Fable gate stays first-class.** The self-audit does NOT replace it — it raises the
floor Fable starts from. The high-effort Fable checkpoints remain the non-negotiable quality mechanism,
now run as cheap verification instead of an expensive climb.

**Phoenix application (proof the firewall is shape-agnostic).** Phoenix's slender scythe wings would
"fail" a naive "must have big wings" reading — but the firewall says *confident MASS*, and Phoenix's
mass lives in the train fan + covert sheet + teardrop quill-vanes, not spider spars. The pre-score
(QA-0) reconciles each invariant to Phoenix's own design, with per-axis round-0 targets ≥ 3.5.

**What it unlocks.** Pearl/Obsidian/Phoenix build sessions start from a PRE-SCORED sheet and a quality
floor, self-audit to ~3.5 before Fable, and pass in 2–3 high-effort gate rounds — the ~5 rediscovery
rounds and the CP2 second campaign are designed out, without homogenizing the roster.
