# 2026-07-09 — PR-C: detonations replace the chord; a rear-cam charge telegraph lives on the silhouette; an arrival-exact speed profile is a position CONTROLLER

**Did / learned.** Owner playtest of the unleash PR-B drove three builds (all `?unleash=v1`-gated, PR #313):
1. **A repeated reward jingle wears out — musical RESOLUTION is for rare moments; a repeated reward wants IMPACT VARIETY.** The finale's tonic-chord ring-out ("nice but airy… annoying to hear consistently") became an N-scaled DETONATION: the kept thwack front ("sounds like ejecting emberwyrms"), then a boom whose depth/length/level scale by the riser's own class knob `big = pow(n/3, riserTickPowN)`, the bossDefeat heavy low pair fading in only at n≥5, `min(n,6)` parity-detuned falling debris shards (shieldShatter idiom), and a full-only crackle tail. Per-strike: plucks + small detonation pops (owner's pick — melody AND ordnance). Instance variety without RNG: a module `detSeq` counter walked through the det idiom (±0.8% pitch, 0–24ms debris timing) — successive volleys never clone, render==live never at risk. DELETING the chord machinery (voices/bus/release/cancel + 3 config keys) also retired the entire hanging-drone bug family — sometimes the fix for "sound X misbehaves in edge cases" is "stop sustaining anything."
2. **A rear chase cam never sees the mouth — put the charge on the shapes it DOES see.** The owner asked "can the torso visibly arch?" Exploration: the cam looks down the dragon's BACK (head faces away); the readable shapes are the wingspan, the dorsal line, and the space around the body. The VISIBLE INHALE is a fuse01-driven combo: torso ARCH (+0.38 rad into the whole-body pitch channel + neck-crane/hip-drop on spine rigs), wing MANTLE (flapSpeed ×(1−0.6·inhale), amp ×0.55, rootFlap biased −0.55 = the whole stroke rides HIGH — one shared-layer line that hits every wing rig), jade GATHER (new `gatherPulse` in particles.js — the public `burst()` can't do inward motion, so it wraps the private `spawn()` with an inward `velBias`, gravity 0, stretch 2.2: sparks streak INTO the launch shoulder where the exhale burst already fires), glow swells (wing emissive + jade lerp — fever pink WINS, Surge's role colour never touched), and a −2 FOV pinch + 0.3 dolly (level channel, reduced-motion gated). Plumbing = the `setDragonLook` pattern: one setter, one damped channel, every injected term ×inhale01 → **byte-identical at 0** (the L245 endpoint law — wingflap/flapcheck/skinnedwing/torsoshoulder/shoulderbridge all green untouched).
3. **An arrival-exact speed profile should TRACK POSITION, not integrate speed.** The lunge (emerge 55% → accelerate to 145%; `lungeProfile [0.55,1.45]`, LAW (p0+p1)/2 === 1) was specced as midpoint-sampled open-loop integration + a float tail-guard — but the first frame integrates before any controller runs (a hidden one-frame error) and variable dt drifts. The robust form: each frame set `vrel = (relExact(t+dt) − rel)/dt` where relExact is the profile's ANTIDERIVATIVE (linear p → closed form), with constant-speed continuation past T. relExact sits strictly below the constant-speed line before T and equals it at/after — so the first frame where rel ≥ target is IDENTICAL by construction: never early, never late, self-correcting under any dt, no fudge factors. Seed spawn vrel at p0·base so frame 1 doesn't pop. T-W8 sweeps 3 depths × 3 dts × 2 profiles + a null reference — all land on the constant-speed frame with monotone rel.

**→ Systematize.** (1) Ask of every repeated reward sound: "will the 50th play still satisfy?" — resolution/melody habituates; impact + deterministic micro-variation doesn't. (2) A charge/state telegraph on a player-controlled actor must be designed for the CAMERA's actual view of it — enumerate the visible shapes first (silhouette > glow > particles > HUD), then drive them all from one eased scalar that is exactly 0 at rest (the endpoint law makes coexistence free). (3) Any "follow a speed curve but arrive on the same frame" mechanic = a position-tracking controller against the curve's antiderivative, never open-loop speed integration. (4) Directional/inward particle effects go through new exported wrappers over the private `spawn()` (the `rollWake` pattern) — don't widen `burst()`.
