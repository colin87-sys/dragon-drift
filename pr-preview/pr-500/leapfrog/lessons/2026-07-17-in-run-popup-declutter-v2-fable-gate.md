# In-run "calm at speed": the harsh critic caught that COALESCING pop-ups is worse than the disease

Supersedes the same-day `2026-07-17-in-run-popup-declutter.md` (that lesson's fix was WRONG — this is why).

**The arc.** Owner: "all the different pop ups as you're playing feel cluttered." First fix: unify every
per-action earn under one coalescing bell key so they merge into a running line. A fresh HARSH FABLE CRITIC
failed it **3.2/5 REVISE** and was dead right — coalescing a running line created a whole class of defects
the headless "one line, one slug" test never surfaced:
- **Dishonest number:** the line kept the LATEST earn's points + a coalesce ×count → `+30 ×5` reads as
  +150; the real sum was +190. Wrong product AND wrong sum.
- **Double-`×N` collision:** skill earns already embed `×streak` in their text (`PERFECT PHASE ×3 +40`),
  and the bell appends its own coalesce `×5` → `PERFECT PHASE ×3 +40 ×5`. The verified test fired only ONE
  perfect so it never showed — a coverage blind spot.
- **Color-law break:** coalesce keeps the FIRST line's role but swaps the text → a gold underline under a
  cyan-role word.
- **Legibility:** the 0.28-alpha plate sits on the bright horizon band; the toast was invisible.

**The real fix (critic's preferred model, re-scored 4.4/5 PASS).** The clutter was never the lane count —
it was **per-action FREQUENCY**. So pull routine earns out of the lane ENTIRELY: `ringPopup`/`nearMiss`/
`roll`/`parry`/`gate`/`thread` become no-ops (signatures kept for the callers), `phasePopup` keeps only the
rare assisted coach-hint. Per-action feedback is carried by the persistent **score counter** + the
**combo multiplier** (escalates live) + the ring-flash. The toast lane is reserved for genuine MOMENTS,
and the chain **resolves into ONE tally** (`+banked · ×peak CHAIN`, `ritual:true` so it never coalesces) —
the "cash out the combo" beat (Hades / Tony Hawk), which is MORE premium than a stream of per-trick toasts.

**Reusable laws.**
1. **A single toast lane still reads as clutter if it fires on every micro-event.** Density is the lever,
   not lane-count. Move high-frequency feedback to a PERSISTENT readout (counter/meter); reserve the
   transient lane for rare moments + a cash-out summary.
2. **Coalescing a running line is a honesty trap.** If you merge, the number MUST be a true sum and carry a
   single meaning — never "latest text + count of mixed things," and never a `+N` next to a gold `×M`
   (that reads as multiplication). Usually it's cleaner not to merge at all.
3. **Verify the WORST case, not the happy path.** The double-`×N` bug only appears when an embedded-`×`
   text coalesces; a 1-perfect-among-5 test hid it. When you add coalescing, test two of the SAME
   `×`-bearing event back to back.
4. **The builder never judges its own output.** A fresh harsh critic at the ≥4.2 bar turned a plausible-
   looking "it collapses to one line" into a REVISE with four concrete blockers in minutes. Run it before
   shipping feedback/juice changes, and RESUME the same critic to re-confirm against its own findings.
