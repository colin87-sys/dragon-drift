# 2026-07-10 — Premium rebuilds need contract-first metadata, not just pretty meshes

**Did / learned.** Implementing the Dawnfire Empress showed that the fastest way to keep a premium procedural dragon honest is to publish the sheet's decisive silhouette dials as testable metadata at the builder boundary. The pyre-train is a visual idea, but the failure classes are numeric: quill count, fan sector, gap proxy, cant balance, glow ladder, and surge-material exclusions.

**Gotcha.** The generic rig only forwards a fixed set of `parts` handles. If a bespoke tail owns a sheet-critical invariant, the model assembler must explicitly adopt that metadata (`pyreTrain`) or the starter test can only infer from roster dials and misses construction regressions.

**Reusable pattern.** For each premium signature, add a tiny metadata object from the hero builder and assert it in the premium's own `tests/starters.mjs` block. Keep it additive/nullable so existing dragons stay byte-identical. This gives the visual gate a pre-flight checklist that catches ring/halo, ladder, and surge-discipline mistakes before screenshot review.

**Unlocks.** Future premium replacements can make their signature measurable without overfitting pixels: author the look in geometry, publish the contract, and let the harsh visual gate focus on beauty/distinctiveness instead of basic compliance.
