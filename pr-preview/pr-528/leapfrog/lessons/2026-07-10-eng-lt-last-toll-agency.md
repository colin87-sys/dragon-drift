# ENG-LT — KNELLGRAVE "The Last Toll" agency rework: un-gating a survival exam (a scoped reversal), a resolve meter as a SECOND resolution, and the one hit that threads an absolute seal

**What we did.** Turned KNELLGRAVE's final phase from a passive wait into an active exam, from **owner playtest feedback** ("you just fly close and occasionally bullets come out you can't even reflect — what am I missing?"). The diagnosis was correct and worth stating plainly: during the survival card the boss is SEALED (chip deflects), the ENG-H toll-wall graze is gated OFF (`setpieceT < 0`), and the spiral tolls are non-reflectable — so for ~26s there was no chip, no graze, no parry, just dodge-and-wait-a-timer. Three integrated mechanics fixed it: (1) **ride mode** — un-gate the toll-wall graze during the Last Toll; (2) a **RESOLVE meter** — riding the walls, parrying the seal-era ambers, and striking the clapper fill a meter that breaks the seal EARLY + staggers the bell; (3) the **clapper seam** — the bound prisoner is the seal's one scoped weak-point. This is the first "make it fun" rework (not a build-queue row) and the first that **reverses a shipped design decision**.

**The protocol variation: two Fable passes, spec-then-verify.** ENG-EW/ENG-H had committed specs I only drift-checked. Here no spec existed, so the flow was: Fable pass 1 **wrote** the drift-checked PRE-BUILD spec (design + live-symbol citations), then Fable pass 2 **independently drift-checked** that written spec (the owner's "make sure it checks it"). Both returned GO-WITH-CORRECTIONS; pass 2 caught only footnote-grade items (below) — but the independent read is what makes "the design agent checked its own work" trustworthy on a change that touches an absolute damage seal. *Reusable: for a from-scratch design, spawn TWO Fable passes — one to author the spec against live code, one to verify it — not one.*

**The overhead-geometry trap the ENG-H lesson pre-flagged, finally paid.** ENG-H gated the graze off during setpieces partly because at the Last Toll's overhead ride `pose.rel ≈ 1.5–4.5`, so the bell mouth's rel resolves to `−0.6…2.4` — behind/at the plane (`resolveEmitOrigins` skips `rel <= 0.5`), and even when resolvable `srel/slow ≈ 0.05–0.2s` → a pocket that dies before one tick at radius ~0.5–2u. **Fix: ride-mode pockets never read `srel` — they're driven off the TOLL CADENCE** (`RIDE_POCKET_DUR = 1.8`, `r1 = SPIRAL_OUT_SPD·dur`). The honesty identity survives because the wavefront's *plane-projected lateral* radius is `SPIRAL_OUT_SPD·t` regardless of rel (bullets expand laterally at 9 u/s invariant of their closing) — so `discR = discR1·(age/dur) = 9·age` still IS the true ring, no cap, no lie. *Reusable: `drawn == wavefront` is a lateral-projection identity, not a 3D-distance one — it survives an overhead camera the naive time-to-plane math can't.*

**Reversing a shipped gate, scoped so it leaks nowhere.** The `setpieceT < 0` purity was load-bearing for every OTHER setpiece. The reversal is fenced by a triple-gated predicate — `discRideMode() = grazeForm==='shrinkDisc' && def.survivalResolve && setpieceDef.id === def.survivalResolve.setpiece` — so it fires ONLY on the one def that carries all three and names `lastToll`; pendulumSweep and every other boss keep the shipped bytes. The shipped ENG-H purity test **twin split**: the sweep leg stayed verbatim law, the lastToll leg was rewritten to assert the ride pocket now ARMS. *Reusable: when reversing a shipped invariant for exactly one case, gate it on a conjunction that only that case satisfies, and split — don't delete — the test that encoded the old invariant.*

**Two ride-mode inversions of ENG-H's own laws (both safe only here).** (a) `discCd = 1.6` is NEITHER consulted NOR set in ride mode — the accelerating knell's ~1.0s gaps would let the cd silently drop pockets exactly when the exam is richest. (b) Replace-on-arm is allowed (ENG-H forbade it) — overhead, each toll's plane-crossing spends itself ~0.1s after the toll, so the superseded ring is no longer a live threat; a fresh toll re-arming at radius 0 makes pockets *tighten* as the knell accelerates (escalation for free). The ENG-H "do not fix the cd / no replace-on-arm" laws stay **station/peal-scoped** (P3's `burst count 2` never runs in ride mode). *Reusable: a graze law derived from "the wall is still flying at the plane" inverts safely once the wall crosses the plane in ~0.1s (overhead) — scope the law to the geometry that birthed it.*

**The seal carve-out: the one hit that threads an absolute return.** `damageBoss`'s survival seal is `if (activeCard.survival) { …; return; }` above every kind check. The clapper seam inserts ABOVE it (after `labPacifist`), scoped FOUR ways so nothing leaks: survival-card-only, **rider-kind only** (`player` returns stay the parry economy's — no double-dip; `surge` stays sealed; `lance` never flies via `lockDeflected`), the LANDING point within `CLAPPER_HIT_R` of `clapperHead`'s live world x/y, AND the PLAYER within `CLAPPER_NEAR`. It moves NO hp — the fairness semantics stay absolute — it only feeds the meter and rings the bell (`model.hurt` → the pre-built ringKick; a diegetic `bellToll(0.3)`, never the deflect `shieldPing`). The verb is **proximity, not aim**: knellgrave declares no `lockParts`, so rider chip always lands at the pose centre — the player-near gate ("dart under the bell") is the skill filter. *Reusable: to make one target hittable through an absolute seal, insert ABOVE the seal and scope by kind + landing-point + player-proximity; feed a meter, never the hp — the seal's fairness contract stays intact.*

**The load-bearing fairness invariant.** The meter is a SECOND, FASTER resolution that calls the SAME `endCard()` hook the timer-outlast path uses (an early hitless break IS a capture — mastery shortens the exam AND banks the card). The timeout branch is untouched: a dodge-only player who never feeds the meter gets the shipped fight verbatim (§5f "no card ever hard-walls a weaker player"). The meter only ever *helps*. *Reusable: add agency to a "just survive" beat as an alternate WIN accelerator through the existing resolution hook — never a second, harder gate.*

**Two test gotchas banked.** (1) **Station geometry:** at the non-ride station pose the clapper hangs only ~1.6–5u below the pose centre, so "a pose-centre landing is far from the clapper" is FALSE at station (it's true only during the ride's dread drop). The landing-gate test had to place the landing point *explicitly* far (`clapper.x + 40`), not rely on the pose centre. (2) **`debugForceCard` bypasses `beginCard`** (drift-check catch): it sets `activeCard`/`cardTimer` but not `cardHits0`/`cardExpired`/`resolveK` — so tests must reset via the per-arm `boss.resetBoss()` (the ENG-EW 2nd-encounter law), never assert beginCard-reset semantics through `debugForceCard`, and run ≥1 `updateBoss` before any synthetic `emit('bossDamage', …)` so `lastPlayer` is populated.
