# 2026-07-14 — A build sheet that becomes a supersede-palimpsest degrades the build: consolidate to ONE contract (and let the owner's reference win the emissive doctrine)

**Did / learned.** Rewrote `TEMPEST-THUNDERHEAD-BUILDSHEET.md` as a single self-contained v3
contract (§1–§10 at the top, mirroring the Revenant §B structure that produced the roster's best
build), demoting the entire v0→v2 stratigraphy (§0–§12 concept → §B v1 → §C carryover → §D wing
patch) to a clearly-marked "do not build from here" appendix. The diagnosis: the Revenant sheet
drove a premium build because its §B was ONE contract read top-to-bottom; the Tempest sheet had
decayed into a PALIMPSEST — a builder had to read §B, patch it with §C, replace parts of it with
§D, and skip superseded fragments scattered through §5/§R/§F/§B.3b–§B.8 (four strata, ~10
supersede banners). Every "read X but ignore Y, use Z instead" is reconciliation work the build
chat does with finite attention, and it does it WORSE than a design chat doing it once. Second
half: the owner's Stormfork reference image shows the lightning as a generous glowing garment
(lit bolt wing-frames, glowing underbelly/chest veins, lit crest, flame-tuft tail) — the sheet's
"withheld, duty 6–18%, dark otherwise" doctrine contradicted the picture. Per the Revenant lesson
(the reference outranks the digest), the doctrine was revised in the owner's favour and RECORDED
as a revision: hum at idle (humFloor 0.30→0.90, breathing), pulseTimer strikes as the PEAK
(contrast capped 2.2–4.0× — no ignition-from-dark), Surge as the break; CHARGING survives whole.
Gotchas worth banking: (1) an always-lit garment costs ZERO extra overdraw if it is opaque
emissive geometry — the real budget is bloom AREA (bounded by a coverage ladder, ≤12% asserted by
strip-area sum), and photosensitivity actually IMPROVES because strike modulation depth shrinks;
(2) revising "withheld" toward "present" moves the nearest-neighbour collision from Vesper to
SOLAR — a doctrine change re-ranks the anti-collision matrix, so the calibration tiles and the
watched-cells table must move with it; (3) retired asserts must be NAMED in the new contract
(the ≤0.06 ember floor, "heart+eyes ≥85%", `feverWing 0x000000`) or a future chat re-adds them
from the appendix.

**→ Systematize.** The pattern generalizes to every long-lived design doc in this repo (boss
sheets, biome sheets, GRAPHICS-OVERHAUL initiatives): amendments may ACCUMULATE as patch sections
during a design cycle, but **before a build cycle starts, consolidate** — one authoritative
contract at the top, all history in an appendix, zero supersede banners on the load-bearing path.
The test is mechanical: *can a build chat produce the creature reading ONE contiguous section?*
If any instruction says "X wins where it conflicts with Y," the sheet has failed the test.
Corollary rule for identity revisions: when an owner reference contradicts a frozen law, the law
is revised IN the contract (with the reason and date), the superseded doctrine is named in
SETTLED so it cannot be re-litigated, and every downstream artifact that encoded the old doctrine
(asserts, calibration tiles, anti-collision cells, overdraw budgets) is re-derived in the same
pass — a doctrine flip done halfway is a new palimpsest.

**→ Leapfrog.** The Tempest build (I0–I5) now starts from a contract as clean as the Revenant's
did, with the garment doctrine pre-reconciled instead of discovered mid-build at a failed gate.
Next moves this unlocks: apply the consolidation test to the remaining three Fresh Five sheets
BEFORE their build slots (Stiletto is next — it already has §R/§F strata); and the
"opaque-emissive garment, coverage-capped" pattern is now a proven-on-paper alternative to both
"withheld seam" and "additive glow shell" for any future always-visible spectacle (Sylph's
curtain, Tocsin's ring afterglow) — geometry-carried light scales to generous coverage without
touching the transparency budget.
